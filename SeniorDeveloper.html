Instructions

The objective of this project is to build a web page that suggests activities to the user based on their preferences.

To do so, you should leverage the boredapi. This api will return one randomized activity at a time for the given preferences. The returned activity should be displayed to the user in an attractive way. The following attributes of the activity should be displayed:

  • Description
  • Type (e.g. "education")
  • Number of participants
  • Relative price
  • Link (if provided)
  • Accessibility rating

Each time the user requests an activity, all previous activities should be collapsed and display only their description. If the user clicks on a collapsed activity, it should expand to show the detail.

The user should be able to specify the following preferences that are then passed to the api:

  • Type: category
  • Number of participants: unsigned integer
  • Relative price: range (free -> expensive)
  • Accessibility rating: range (most -> least)

You can use any libraries, frameworks, languages, etc. that you would like as long as the end result is a web page. You may use any written or web resources but if you sample code please cite all uses in the comments. You may not get help from other people - we want to see your code.

The final product will be assessed in three areas in order of importance:

  1. Functionality as defined by specification
  2. Ease of use and workflow clarity
  3. Styling, responsiveness and structure
